## 🎯 認識 Go Strategy App

### 什麼是 Go Strategy App？

**Go Strategy App** 是一款搭載世界頂尖圍棋 AI「KataGo」的圍棋策略分析工具。無論您是剛學會下棋的新手，還是追求段位突破的業餘高手，這款 App 都能為您提供專業級的落子建議和局面分析。

### 核心特色

| 特色 | 說明 | 適用情境 |
|------|------|----------|
| 🎯 **AI 推薦棋步** | 顯示當前局面下的前三名最佳落子位置 | 對局中猶豫不決時 |
| 📊 **即時勝率分析** | 每一手棋的勝率變化即時呈現 | 檢討棋局、分析得失 |
| 💾 **智能快取系統** | 開局庫 + 本地快取，分析快如閃電 | 快速取得常見局面的建議 |
| 📱 **完全離線運作** | 無需網路，AI 在您的裝置上運算 | 通勤、旅行、無網路環境 |
| 🔢 **多種棋盤規格** | 支援 9x9、13x13、19x19 路棋盤 | 從練習到正式對局 |
| ☁️ **雲端同步** | 棋譜自動備份到 Google Drive/iCloud | 跨裝置使用、資料不遺失 |

### 支援平台

✅ iOS (iPhone, iPad)
✅ Android 手機與平板
✅ macOS (Apple Silicon 與 Intel)
✅ Windows (即將推出)
✅ 網頁版 (任何瀏覽器)

### 操作 1：落子

#### 如何下棋

1. **點擊空白交叉點**
   - 用手指（觸控螢幕）或滑鼠點擊棋盤上的交叉點
   - 黑棋與白棋會自動輪流

2. **系統自動檢查**
   - ✅ 合法位置：棋子立即出現
   - ❌ 違規位置（禁著點、已有棋子）：顯示提示訊息

3. **AI 自動分析**
   - 落子後，AI 會立即開始分析

| 🟠 **橘色** | 第 3 名 | **可接受** | 可下但略遜一籌（<3 目差距） |

#### 分析面板資訊

點擊「推薦棋步」頁籤，查看詳細數據：

```
🔵 1. E5                        ← 落子位置
   Win: 52.3%  Lead: +0.5      ← 勝率與領先目數
   Visits: 150v                ← AI 運算次數
   Source: Book                ← 分析來源

🟢 2. D4
   Win: 51.8%  Lead: +0.3
   Visits: 150v
   Source: Book

🟠 3. F6
   Win: 50.2%  Lead: +0.1
   Visits: 150v
   Source: Book
```

**數據說明：**
- **Win (勝率)**：下這步棋後，該方的預估勝率
- **Lead (領先目數)**：正數表示領先，負數表示落後
- **Visits (訪問數)**：AI 運算的次數，越高代表越可信
- **Source (來源)**：資料來源（Book=開局庫、Cache=快取、Local=本地運算、Live=遠端 API）

### 操作 4：清空棋盤

#### 如何清空

1. **點擊「Clear」按鈕**（或按 Ctrl+N / Cmd+N）
2. 系統會彈出確認對話框
3. 確認後，棋盤恢復空白狀態

⚠️ **注意：** 清空棋盤會清除所有棋步記錄，此操作無法復原！

---

### 操作 5：重新分析

#### 何時使用重新分析？

- 想獲得更深入的 AI 運算結果
- 調整了「Compute Visits」設定後
- 懷疑快取資料不夠新

#### 如何重新分析

1. **點擊「Re-analyze」按鈕**（或按 Ctrl+R / Cmd+R）
2. AI 會忽略快取，重新運算當前局面
3. 新的分析結果會取代舊資料

## 🤖 AI 分析功能詳解

### AI 如何運作？

Go Strategy App 使用**離線優先策略**，按以下順序取得分析結果：

```
使用者落子
    ↓
┌─────────────────────────────────────┐
│ 第 1 層：查詢開局庫 (Opening Book)   │
│ • 內建 3 萬+ 常見局面                │
│ • 速度：⚡ 瞬間 (<0.1 秒)            │
│ • 若找到 → 返回結果 ✓                │
└─────────────────────────────────────┘
    ↓ 未找到
┌─────────────────────────────────────┐
│ 第 2 層：查詢本地快取 (Local Cache)  │
│ • SQLite 資料庫儲存過去分析結果      │
│ • 速度：⚡⚡ 極快 (<0.5 秒)          │
│ • 若找到 → 返回結果 ✓                │
└─────────────────────────────────────┘
    ↓ 未找到
┌─────────────────────────────────────┐
│ 第 3 層：啟動本地 KataGo (Local AI)  │
│ • 在您的裝置上執行 KataGo            │
│ • 速度：⚡⚡⚡ 快 (2-10 秒)          │
│ • 運算完成後存入快取                 │
│ • 返回結果 ✓                         │
└─────────────────────────────────────┘
    ↓ 若離線失敗
┌─────────────────────────────────────┐
│ 第 4 層：遠端 API (Cloud Fallback)   │
│ • 需要網路連線                       │
│ • 備用方案，極少使用                 │
└─────────────────────────────────────┘
```

### 讀懂 AI 的數據

#### Win% (勝率)

**意義：** 下這步棋後，該方的預估勝率

```
🔵 1. E5
   Win: 52.3%  ← 黑棋下 E5 後，黑棋預估有 52.3% 機會獲勝
```

**解讀技巧：**
- **50% 左右**：局勢均衡，雙方勢均力敵
- **> 55%**：該方局面有利
- **> 65%**：該方大優勢
- **< 45%**：該方局面不利

⚠️ **開局迷思破解：**
由於貼目（白棋補償 7.5 目），黑棋在開局時的勝率通常只有 **48-49%**，這是正常現象，不代表下錯了！

#### Lead (領先目數)

**意義：** 預估的領先或落後目數

```
🔵 1. E5
   Lead: +0.5  ← 下 E5 後，黑棋領先 0.5 目
```

**解讀技巧：**
- **+1.5**：領先 1.5 目（微優）
- **+5.0**：領先 5 目（明顯優勢）
- **-2.0**：落後 2 目（略遜）
- **0.0**：完全均衡

#### Visits (訪問數)

**意義：** AI 進行的蒙地卡羅樹搜尋 (MCTS) 次數

```
🔵 1. E5
   Visits: 150v  ← AI 模擬了 150 次對局
```

**解讀技巧：**
- **< 50v**：快速估算，可能不夠精準
- **50-200v**：日常分析，平衡速度與準確度
- **200-500v**：深入分析，適合複雜局面
- **> 500v**：專業級運算，準確度極高

### 對稱感知快取技術

App 使用**對稱感知快取**，大幅提升效率：

```
以下 8 種局面在 AI 看來是「相同的」：

原始局面  旋轉90°   旋轉180°  旋轉270°
   ●         ●         ●         ●
  ○         ○         ○         ○

水平翻轉  垂直翻轉   對角翻轉1  對角翻轉2
   ●         ●         ●         ●
  ○         ○         ○         ○
```

**效果：** 快取命中率提升 **8 倍**，讓 App 更快回應！

**功能：** 設定從開局庫/快取讀取資料的最低訪問數門檻

```
滑桿範圍：100 ─────●──────── 5000

設定值越高 → 只接受深度分析過的資料
設定值越低 → 快速取得結果，但可能不夠精準
```

**建議設定：**
- **100-200**：日常使用（推薦）
- **300-500**：複雜局面、檢討棋局
- **1000+**：專業分析

**範例：**
- 設定 Lookup Visits = 200
- 若快取中的資料只有 150v → 系統會啟動本地 AI 重新運算
- 若快取中的資料有 500v → 直接使用快取資料 ✓

---

### 3. Compute Visits (計算強度)

**功能：** 設定本地 AI 即時運算時使用的訪問數

```
滑桿範圍：10 ─────●──────── 200

設定值越高 → 分析越準確，但越慢
設定值越低 → 分析較快，但可能略遜
```

**建議設定（依裝置效能）：**

| 裝置類型 | 建議值 | 分析時間 |
|----------|--------|----------|
| **旗艦手機** (iPhone 15 Pro, 三星 S24) | 100-150 | 3-5 秒 |
| **一般手機** (iPhone 12, 中階 Android) | 50-80 | 5-8 秒 |
| **入門手機** | 20-30 | 8-15 秒 |
| **平板/電腦** | 150-200 | 2-4 秒 |

**範例：**
- 設定 Compute Visits = 50
- 當需要本地運算時，AI 會進行 50 次 MCTS 模擬
- 結果會自動存入快取，下次遇到相同局面時直接讀取

---

### 4. 貼目 (Komi)

**功能：** 設定白棋的補償分數

**原理：** 圍棋中黑棋先下，具有優勢，因此白棋需要補償分數以平衡局勢。

**常用設定：**
- **7.5**：現代正式比賽標準（中國規則、日本規則）
- **6.5**：部分舊規則
- **0.5**：讓子棋（黑棋讓先）
- **5.5**：黑棋讓先半子

## ❓ 常見問題與解決方案

### Q1: AI 說我下的棋「只有 48% 勝率」，是不是下錯了？

**A:** 不是！這是正常現象。

由於貼目（白棋補償 7.5 目），黑棋在開局時的勝率通常只有 **48-49%**。這是規則設計使然，不代表您下錯了。

**範例：**
```
開局第一手：黑 Q16
AI 顯示：Win: 48.7%  Lead: -0.2

解讀：黑棋略微不利，因為白棋有 7.5 目補償
→ 這是正常的，繼續下就對了！
```

---

### Q2: AI 分析很慢怎麼辦？

**A:** 可以嘗試以下方法：

1. **降低 Compute Visits**
   - 設定 → Compute Visits → 調整到 30-50
   - 犧牲一點準確度，換取更快的速度

2. **使用較小的棋盤**
   - 9x9 棋盤比 19x19 快 **5-10 倍**
   - 適合快速練習

3. **檢查裝置效能**
   - AI 運算需要一定的運算能力
   - 入門手機建議使用 Compute Visits ≤ 30

4. **清除快取（罕見情況）**
   - 設定 → 資料管理 → 清除快取
   - 可能解決資料庫肥大導致的慢速

### Q5: AI 的建議一定是對的嗎？

**A:** AI 非常強大，但不是絕對正確。

**KataGo 的等級：**
- 職業九段水平（超越絕大多數人類棋手）
- 但仍有盲點，特別是極罕見的局面

**使用建議：**
- ✅ **參考 AI，但不要盲從**
- ✅ **理解 AI 的思路**：為什麼推薦這步？
- ✅ **結合自己的判斷**：AI 不會考慮對手心理、時間壓力等因素

**範例：**
```
AI 推薦：E5 (Win: 52%)
您下了：D4 (Win: 51.8%)

→ 只差 0.2%，幾乎沒有區別！
→ 您的棋也是好棋 ✓
```

### Q6: 為什麼有時候 AI 標記閃爍或消失？

**A:** 這是正常的分析流程。

**分析階段：**
```
階段 1: 查詢快取（瞬間）
   ↓
   若未找到 → 移除舊標記
   ↓
階段 2: 啟動本地 AI（2-10 秒）
   ↓
   顯示「分析中...」進度條
   ↓
階段 3: 顯示新的 AI 標記
```

**優化建議：**
- 若經常遇到未快取的局面，可提高 Compute Visits 以加快分析速度
- 或降低 Lookup Visits，接受快取中品質較低的資料

### 圍棋術語

| 術語 | 說明 |
|------|------|
| **貼目 (Komi)** | 白棋的補償分數，通常為 7.5 目 |
| **禁著點 (Ko)** | 不能立即提回的棋子位置 |
| **死活 (Life and Death)** | 判斷棋子是否能存活的技巧 |
| **星位** | 棋盤上的標示點，常見的第一手落子位置 |
| **手數 (Move Number)** | 棋局中的第幾手棋 |
| **勝率 (Win Rate)** | 預估的獲勝機率 |
| **目 (Point)** | 圍棋計算勝負的單位 |

### AI 術語

| 術語 | 說明 |
|------|------|
| **KataGo** | 世界頂尖的開源圍棋 AI |
| **MCTS** | 蒙地卡羅樹搜尋，AI 使用的演算法 |
| **Visits** | AI 模擬對局的次數 |
| **對稱性 (Symmetry)** | 利用棋盤對稱特性優化快取 |
| **Zobrist Hash** | 棋盤狀態的雜湊值，用於快速比對 |

### 技術術語

| 術語 | 說明 |
|------|------|
| **快取 (Cache)** | 儲存過去分析結果，加快查詢速度 |
| **開局庫 (Opening Book)** | 預先運算的開局資料庫 |
| **離線優先 (Offline-First)** | 優先使用本地資源，無需網路 |
| **SGF** | 圍棋棋譜的標準檔案格式 |
| **GTP** | 圍棋文字協定，用於與 AI 溝通 |
